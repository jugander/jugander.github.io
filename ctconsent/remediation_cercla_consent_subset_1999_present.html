How This List Was Built

Database accessed: Connecticut Open Data (Socrata), DEEP formal enforcement datasets:

  • tvbr-6b6t (Formal Enforcement Case Summaries 1999-2021)
  • t2bf-45ba (Formal Enforcement Case Summaries 2021-Present)

Rough database stats from local snapshots used for this build:

  • tvbr-6b6t: records 2070; with PDF URL 586; missing PDF URL 1484; with order metadata 586
  • t2bf-45ba: records 210; with PDF URL 210; missing PDF URL 0; with order metadata 209
  • Combined: records 2280; with PDF URL 796; missing PDF URL 1484; with order metadata 795

How queried: rows were exported from each dataset's API (/api/views/<id>/rows.json?accessType=DOWNLOAD) and filtered to consent actions (consent in enforcement-action field).

Inclusion rule: consent-action rows were included if at least one condition matched:

  • Consent-action keyword in enforcement-action field (case-insensitive): consent
  • Program keyword in program field (case-insensitive): site remediation
  • Remediation-standards text keywords/patterns (case-insensitive): remediation standard regulation, remediation standards regulation, rsr, 22a-133k
  • CERCLA text keywords/patterns (case-insensitive): cercla, comprehensive environmental response, compensation, and liability act, and U.S. Code citation variants matching 42 U.S.C. ยง 9600-9609

Possible shortcomings:

  • This filter is metadata-based; it does not full-text search every PDF body.
  • Keyword matches can miss relevant orders if wording differs, contains typos, uses uncommon abbreviations, or appears only in scanned/OCR-imperfect PDFs.
  • These two datasets are not a complete legal index of all consent orders. Based on English Station validation, both partial and non-partial consent orders can be absent from dataset rows.
  • Some orders are publicly posted on CT.gov in other locations (for example, adjudication/complaint exhibits or AG press-release attachments) instead of appearing in the DEEP enforcement datasets.
  • Some source rows have no pdf_url or no order number, so records can be real but not directly downloadable from dataset metadata.
  • Order-number patterns and expected portal URL patterns are not fully reliable. A record identifier may exist without a matching public enforcement URL, and numbering can be non-contiguous.
  • Coverage and links depend on DEEP/Open Data publication quality and can lag updates or include broken links over time.
